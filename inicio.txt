****Interfaces del CPE original
Sabemos que existen:
eth0   → red Kubernetes / control plane
net1   → salida WAN / Internet (10.100.1.1/24)
net2   → interfaz multus auxiliar
brint  → bridge interno (192.168.255.254/24)
axscpe → interfaz hacia access/OVS
Eso es importantísimo porque FRR tendrá que reproducir exactamente esa estructura.

Configuración funcional conocida
Sabemos que el CPE original funciona con:
Ruta default
default via 10.100.1.254 dev net1
Red LAN
10.20.1.0/24 via 192.168.255.253 dev brint
NAT
MASQUERADE out net1
FORWARD brint → net1
RELATED,ESTABLISHED net1 → brint
Forwarding
net.ipv4.ip_forward = 1
